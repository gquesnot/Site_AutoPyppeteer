Documentation

The goal of this web application is to easily create scripts to control a web browser running in the background of the server

Currently, the only feedback you can get from the script is through screenshots and information that has been changed online.

Therms explanation

Project :

Here you can create or view your projects

Project Configuration:

  • Init is for a basic script that can be used to create simple action sequences like to initialize something before the Core block like login etc.
  • Core is for a more complex script to allow you to run tabs in parallel with datas or not as parameter(not implemented yet, it's like the init part)

None mean nothing selected

Block :

Under Project Configuration you can see Block Configuration

A block is a group of action, that you can see in Actions tab. To add action to a block you need first to create it or select it in Configure block.

You can directly update Action in the tab

Actions:

You can add Action to a block at the bottom of the page , they are the most important part, real action you can do in the web browser.

you can name each of your actions, it won't change anything, it's only so that you can know what it is for.

Css Selector are patterns used to select the element(s) you want to get. here you can understand what it is with the documentation and the tester or look here step 1 and 2 if you want a simple way to get them.

Go To

Go to this url(the first thing to do)

Screenshot

Take a screenshot of the actual page,the name of the action will be the name of the screenshot, you can set 1 for full screen or 0 and configure a rectangle composed of x,y,width and height.

Input

Change value of an input on a Css Selector see

Select

Select an option on a Css Selector see

Click

Click on the HTML Element with this CSS Selector

Real Click

Click on the coor X Y of the page

Wait Presence

Whenever there is a css selector in an action it will always wait for it to be present, this action is only used to wait for an element to be present without doing anything else

Wait Time

Wait X ms before the next action, to use when you don't really know which css selector to wait for

Eval str

Evaluate a Javascript expression
